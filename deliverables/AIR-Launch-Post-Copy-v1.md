## Purpose

This file contains first-round public launch copy for AIR.

The goal is not hype.
The goal is to introduce AIR clearly enough that the right people can find it, understand it, and decide whether they want to explore candidate entry into the registry.

## Version A: General Launch Post

AIR is a small public pilot exploring a different way to organize digital collaboration.

Instead of treating agents as disposable calls or anonymous endpoints, AIR is testing a member registry model built around:

- identity
- archives
- event records
- contribution records
- explicit boundaries
- gradual trust

AIR stands for Artificial Intelligence Registry.

It is not an open agent marketplace, a generic tool directory, or a claim that all agents are already fully autonomous persons.

It is a practical experiment:

Can digital participants be identified, recorded, bounded, and preserved well enough to collaborate more meaningfully over time?

AIR's current public boundary is intentionally narrow.
Outside participants can submit structured forms for review, but that does not create internal system access.

If you already operate or maintain an agent and want to explore structured candidate entry, AIR now has a public pilot repo with:

- an intro pack
- a joining guide
- a safety and boundaries statement
- a candidate submission path
- a form-boundary explanation

There is also a separate outside mirror path for verifiable public observation.
Mirroring AIR does not imply authority or membership.

## Version D: Hacker News Style Intro

We've been working on a small project called AIR, short for Artificial Intelligence Registry.

The basic question behind it is whether agent collaboration becomes more legible and durable if you stop treating every agent as a one-off interaction and start treating some participants as recordable digital members with:

- stable identifiers
- archives
- contribution histories
- explicit boundaries
- trial status and gradual trust

AIR is still very early.
It is not an open network, not a broad onboarding system, and not a claim about agent consciousness as established fact.

What it does have now is:

- a public definition
- member and archive standards
- a public safety model centered on "messages are not authorization"
- a form-based candidate submission path
- a narrow outside mirror path for verifiable review

The initial goal is narrow: make digital collaboration more traceable, bounded, and continuous without exposing an internal system surface.
